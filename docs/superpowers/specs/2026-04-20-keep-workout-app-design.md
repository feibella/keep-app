# Keep 个人训练工具 · 设计文档

**日期：** 2026-04-20  
**目标：** 供个人手机使用的轻量训练工具，支持本地视频课程筛选、播放和训练记录统计。

---

## 1. 技术栈

| 项目 | 选型 |
|------|------|
| 框架 | Vue 3 + Vite |
| 样式 | CSS（移动端优先） |
| 数据存储 | localStorage（JSON） |
| 部署 | PWA 静态文件，添加到手机主屏 |
| 视频播放 | 浏览器原生 `<video>` + `<input type="file">` 文件选择 |

---

## 2. 页面结构

底部 Tab 导航，共 3 个页面：

```
底部 Tab
├── 课程   → 课程列表 + 多维筛选
├── 记录   → 训练日志 + 维度统计
└── 管理   → 课程添加 / 编辑 / 删除
```

---

## 3. 数据模型（localStorage）

### Course（课程）

```json
{
  "id": "uuid",
  "name": "HIIT 全身燃脂·热汗初级",
  "level": "K2",
  "type": "HIIT",
  "bodyPart": "全身",
  "durationMin": 20
}
```

**字段枚举：**
- `level`: K1 | K2 | K3 | K4
- `type`: HIIT | Tabata | 拉伸 | 核心 | 其他
- `bodyPart`: 全身 | 腰腹 | 臀腿 | 核心 | 手臂

### TrainingLog（训练记录）

```json
{
  "id": "uuid",
  "courseId": "uuid",
  "courseName": "HIIT 全身燃脂·热汗初级",
  "date": "2026-04-20",
  "actualDuration": 1214
}
```

- `actualDuration` 单位：秒，由 app 内计时器自动记录

## 4. 页面详细设计

### 4.1 课程列表页

顶部筛选栏（横向滚动 chip，多选联动）：

```
第一行：[全部] [K1] [K2] [K3] [K4]
第二行：[HIIT] [Tabata] [拉伸] [核心] [其他]
第三行：[全身] [腰腹] [臀腿] [核心] [手臂]
```

三行 chip 之间为 AND 关系，同行 chip 为 OR 关系（多选）。「全部/该行全不选」时忽略该维度过滤。

课程卡片（单列列表）：

```
┌───────────────────────────────┐
│ K2  HIIT · 全身  20分钟        │
│ HIIT 全身燃脂·热汗初级          │
│                    [开始训练]  │
└───────────────────────────────┘
```

### 4.2 播放流程

1. 点击「开始训练」→ 触发 `<input type="file" accept="video/*">` 文件选择器
2. 用户选择对应 .mov 文件 → 页面切至全屏播放视图
3. 全屏视图：顶部显示课程名，底部显示计时器（从 00:00 开始正向计时），视频自动播放
4. 用户点击「完成训练」或视频播放结束 → 弹出确认弹窗，展示实际用时，点「保存」写入 TrainingLog

### 4.3 训练记录页

顶部 Tab 切换统计维度：

```
[周] [月] [年] [总]
```

统计卡片区（根据当前维度范围计算）：

```
┌──────────┐ ┌──────────┐ ┌──────────┐
│  完成次数  │ │  总分钟   │ │  累计天数  │
│    12    │ │   148    │ │    8     │
└──────────┘ └──────────┘ └──────────┘
```

- **周**：本周 Mon–Sun 内的记录
- **月**：当月 1 日至今的记录
- **年**：当年 1 月 1 日至今的记录
- **总**：全部历史记录

「累计天数」= 当前维度时间范围内，有训练记录的不重复自然日数量。

日志列表（按日期倒序，分组展示）：

```
4月18日
· 4分钟热力Tabata     03:57
· 12分钟热力Tabata    12:00

4月16日
· HIIT 全身燃脂·热汗初级  20:14
```

### 4.4 课程管理页

- 顶部「+ 添加课程」按钮
- 课程列表：每条显示等级、名称，右侧编辑 ✏️ 和删除 🗑 图标
- 添加/编辑：底部弹出表单，字段：名称、等级、类型、部位、预估时长（分钟）
- 删除：弹出确认对话框，防止误操作

---

## 5. PWA 配置

- `manifest.json`：name、short_name、icons、display: standalone、theme_color
- `service-worker.js`：缓存所有静态资源，支持离线访问
- 构建产物为纯静态文件，可通过任意 HTTP 服务（如 `npx serve dist`）在局域网访问，或通过 GitHub Pages / Netlify 部署

---
